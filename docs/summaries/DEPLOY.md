# מדריך פריסה — לומדת ענף 71

מדריך זה מיועד לצוות שמתקין את המערכת על שרת חדש. בצע את השלבים לפי הסדר.

## דרישות מוקדמות
- **Node.js** גרסה 18 ומעלה
- **PostgreSQL** גרסה 14 ומעלה

---

## 1. הכנת בסיס הנתונים
צור בסיס נתונים ריק ב-PostgreSQL (למשל `rf_learning`) ומשתמש עם הרשאות עליו.
אין צורך להריץ קבצי SQL ידנית — המערכת מריצה את כל המיגרציות אוטומטית בעלייה.

> ⚠️ חשוב: תיקיית `database/` (עם קבצי ה-`*.sql`) **חייבת להישלח יחד עם הקוד** לשרת.
> המערכת קוראת אותם מהדיסק בזמן ריצה. הם אינם נכללים בתוצר ה-build.

---

## 2. הגדרת השרת (Backend)
```bash
cd backend
npm install
cp .env.example .env      # ואז ערוך את .env
```
מלא ב-`.env` את הערכים האמיתיים. **חובה** להגדיר:
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` — פרטי בסיס הנתונים
- `JWT_SECRET`, `REFRESH_TOKEN_SECRET` — מחרוזות אקראיות ארוכות. צור כל אחת ע"י:
  ```bash
  node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
  ```
- `CORS_ORIGIN` (מומלץ בייצור) — כתובת האתר, למשל `https://learn.your-org.co.il`

> אם חסר משתנה חיוני, השרת **יסרב לעלות** ויציג בדיוק מה חסר — זו התנהגות מכוונת.

בנייה והרצה:
```bash
npm run build      # יוצר dist/
npm start          # מריץ node dist/server.js — המיגרציות רצות אוטומטית לפני העלייה
```

---

## 3. הגדרת הלקוח (Frontend)
```bash
cd frontend
npm install
npm run build      # יוצר dist/ — קבצים סטטיים
```
הגש את תיקיית `frontend/dist` דרך שרת ווב (nginx / Apache / כל שירות סטטי),
והפנה קריאות `/api` לשרת ה-Backend (פורט 3000 כברירת מחדל).

---

## 4. ⚠️ הקמת המנהל הראשי (חובה!)
בעלייה הראשונה על שרת נקי, המערכת יוצרת **מנהל זמני (ממלא מקום)**:

| שם משתמש | סיסמה זמנית |
|-----------|-------------|
| `OA000000000` | `Admin123` |

**מנהל זה זמני בלבד ואסור להשאירו.** בצע מיד:
1. התחבר עם המנהל הזמני.
2. צור מנהל אמיתי משלך עם שם משתמש בתבנית `OA` + מספר תעודת זהות.
3. התחבר עם המנהל החדש שיצרת.
4. **מחק את המנהל הזמני** (`OA000000000`) דרך מסך ניהול המשתמשים.

> כל עוד המנהל הזמני קיים, קיימת דלת אחורית עם סיסמה ידועה. אל תדלג על שלב 4.
