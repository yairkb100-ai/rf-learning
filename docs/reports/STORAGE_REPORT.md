# דוח משקל ואחסון — לומדת ענף 71

**תאריך הפקה:** 2026-07-01 · **גרסה:** 1.0 · **נוצר על ידי יאיר כהנא**

מסמך זה מסכם עבור ספק האחסון את משקל המערכת, קבצי ההפעלה שהיא מכילה, ואומדן שטח האחסון הנדרש להרצתה.

---

## 1. סקירה כללית

המערכת בנויה משלושה רכיבים:

| רכיב | טכנולוגיה | תפקיד |
|------|-----------|-------|
| **Backend** (שרת) | Node.js + Express + TypeScript | API, אימות, לוגיקה |
| **Frontend** (לקוח) | React + Vite (TypeScript) | ממשק המשתמש (אתר) |
| **Database** (בסיס נתונים) | PostgreSQL | אחסון תוכן, משתמשים והתקדמות |

הלקוח (Frontend) מקומפל לקבצים סטטיים (HTML/CSS/JS) שמוגשים לדפדפן. השרת רץ כתהליך Node.js. בסיס הנתונים רץ כשירות PostgreSQL נפרד.

---

## 2. קבצי הפעלה (Executables / Entry Points)

המערכת **אינה** קובץ `.exe` יחיד — היא אפליקציית ווב הרצה על Node.js. "קבצי ההפעלה" הם:

| קובץ | גודל | תפקיד |
|------|------|-------|
| `backend/src/server.ts` | ~2 KB | נקודת הכניסה של השרת (מאזין על פורט 3000) |
| `frontend/src/main.tsx` | ~0.5 KB | נקודת הכניסה של הלקוח |
| `START.bat` | ~2 KB | מפעיל נוחות ל-Windows (מריץ שרת+לקוח) |
| `START_LEARNING_SYSTEM.py` | ~3.4 KB | מפעיל נוחות חלופי (Python) |

**חשוב:** קבצי ה-`.bat` וה-`.py` הם נוחות לפיתוח מקומי בלבד. בסביבת אחסון (production) ההרצה מתבצעת דרך `node dist/server.js` (השרת) והגשת תיקיית `frontend/dist` ע"י שרת ווב.

הרצה בפועל דורשת שתהיה מותקנת **סביבת ריצה של Node.js** (גרסה 18+) ו-**PostgreSQL** על השרת. אלה אינם חלק ממשקל הפרויקט אלא תשתית של הספק.

## 3. משקל המערכת — פירוט מדוד

### 3.1 קוד המקור (Source) — מה שנכתב בפועל

| רכיב | גודל |
|------|------|
| קוד השרת (`backend/src`) | 73 KB |
| קוד הלקוח (`frontend/src`) | 166 KB |
| סקריפטים של בסיס הנתונים (`database`) | 15 KB |
| **סה"כ קוד מקור** | **~254 KB (רבע מגהבייט)** |

קוד המקור עצמו זעיר. זהו הנכס האמיתי של המערכת.

### 3.2 תלויות (node_modules) — ספריות צד-שלישי

| רכיב | גודל |
|------|------|
| תלויות השרת | 32 MB |
| תלויות הלקוח | 98 MB |
| **סה"כ תלויות** | **~130 MB** |

תיקיות אלה **נוצרות מחדש אוטומטית** בפקודה `npm install` ואינן חייבות להיות מאוחסנות/מגובות — הן נגזרות מקובץ `package.json` בן כמה קילובייטים.

### 3.3 תוצר הבנייה (Build) — מה שמוגש למשתמש בפועל

| רכיב | גודל |
|------|------|
| לקוח בנוי (`frontend/dist`) | 357 KB (JS דחוס ~105 KB) |
| שרת בנוי (`backend/dist`) | 79 KB |
| **סה"כ תוצר בנייה** | **~436 KB** |

זהו החומר שמוגש בפועל לדפדפן — קל מאוד ומהיר לטעינה.

### 3.4 קבצים מצורפים (Uploads)

| סוג | כמות | גודל |
|-----|------|------|
| דיאגרמות SVG (של המערכת) | 18 | 41 KB |
| קובצי PDF/DOCX שהועלו | 4 | ~14 MB |
| **סה"כ נוכחי** | 22 | **~14 MB** |

> **הערה חשובה:** 13 מ-14 המגהבייט הם קובץ Word (`.docx`) בודד שהועלה לבדיקה. דיאגרמות הלימוד של המערכת עצמן (18 קובצי SVG) שוקלות **41 KB בלבד**. תיקיית ה-uploads תגדל בהתאם לכמות קובצי הלימוד שיצורפו בעתיד.

### 3.5 בסיס הנתונים (PostgreSQL)

| מדד | ערך נוכחי |
|-----|-----------|
| גודל בסיס הנתונים | ~9.3 MB |
| קורסים | 4 |
| פרקים | 21 |
| פריטי תוכן לימודי | 48 |
| שאלות מבחן | 43 |
| משתמשים | 4 |

בסיס הנתונים כולל תקורה קבועה של PostgreSQL (~8 MB). התוכן עצמו קטן; הגידול העיקרי צפוי מרישום תלמידים ותיעוד התקדמותם.

## 4. אומדן אחסון נדרש לספק

### 4.1 מינימום להרצה (Deployment)

| פריט | גודל |
|------|------|
| תוצר בנייה (frontend + backend) | 0.5 MB |
| תלויות production (backend בלבד) | ~30 MB |
| בסיס נתונים ראשוני | ~10 MB |
| קבצי לימוד (SVG בלבד) | 0.05 MB |
| **סה"כ מינימלי** | **~40 MB** |

### 4.2 אומדן מומלץ (עם מרווח לצמיחה)

| פריט | אומדן |
|------|-------|
| אפליקציה + תלויות | 150 MB |
| בסיס נתונים (עד ~500 תלמידים עם התקדמות) | 100 MB |
| קבצי לימוד (מסמכים, תמונות, PDF) | 500 MB – 2 GB |
| לוגים וגיבויים | 200 MB |
| **סה"כ מומלץ להקצאה** | **1 GB – 3 GB** |

### 4.3 סיכום ההמלצה

> **הקצאת אחסון של 5 GB** מספקת בנוחות רבה עבור המערכת, בסיס הנתונים, וכמות סבירה של קובצי לימוד וגיבויים — עם מרווח צמיחה משמעותי.
>
> אם מתוכננת העלאה מסיבית של סרטוני וידאו או מסמכים כבדים — יש להעריך את שטח קובצי המדיה בנפרד לפי הצורך.
